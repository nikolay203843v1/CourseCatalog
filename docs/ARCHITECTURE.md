# Архитектура

## Зависимости модулей

```text
                         ┌─────────────┐
                         │     app     │
                         └──────┬──────┘
                    ┌───────────┴───────────┐
                    ▼                       ▼
             feature:auth             feature:main
                    │                       │
                    └───────────┬───────────┘
                                ▼
                         domain:courses
                                ▲
                                │ implements
                         data:courses

             core:ui подключается только UI-модулями
```

`app` является composition root: создаёт Koin и связывает реализации с
интерфейсами. Функциональные модули не создают Retrofit или Room и не знают,
откуда приходят данные.

## Поток данных

```text
Google Drive JSON
       │
       ▼
Retrofit DTO ── mapper ──► Room Entity ── Room Flow ──► Domain Course
                                                               │
                                                               ▼
                                                         ViewModel StateFlow
                                                               │
                                                               ▼
                                                        Fragment / Adapter
```

Room выбран единственным источником данных для UI. После запуска
`HomeViewModel` инициирует обновление из сети, репозиторий сохраняет ответ в
одной транзакции, а экран автоматически получает новое значение `Flow`.
Благодаря этому уже сохранённый список остаётся доступным при временной ошибке
сети.

## Сохранение избранного

Перед обновлением репозиторий читает локальные значения `isFavorite`. При
маппинге сетевого ответа локальное значение имеет приоритет над полем
`hasLike` сервера. Поэтому повторная загрузка JSON не отменяет выбор
пользователя.

Переключение выполняется одним SQL-запросом:

```sql
UPDATE courses
SET isFavorite = CASE WHEN isFavorite = 1 THEN 0 ELSE 1 END
WHERE id = :courseId
```

Это атомарнее последовательности «прочитать — инвертировать — записать».

## MVVM и состояние экрана

Фрагменты отвечают только за отображение, обработку нажатий и lifecycle.
ViewModel объединяет:

- поток курсов из Room;
- выбранный порядок сортировки;
- состояние сетевой загрузки.

Результат публикуется как неизменяемый `StateFlow<CourseListUiState>`.
Подписка запускается через `repeatOnLifecycle(STARTED)`, поэтому UI не
обновляется после уничтожения View.

## Навигация

Есть два графа:

- корневой `app_nav_graph` переключает авторизацию и основную часть;
- вложенный `main_nav_graph` управляет вкладками нижней навигации.

При успешном входе login удаляется из back stack. Нажатие «Назад» не возвращает
пользователя на форму авторизации.

## Компромиссы задания

- У JSON нет ссылок на изображения, поэтому баннеры являются локальными
  ресурсами и выбираются по id курса.
- Поиск и фильтр статичны по условиям задания.
- Реальной серверной авторизации нет; форма только проверяет введённые данные.
- `Dispatchers.IO` передаётся в репозиторий через конструктор. В production
  проекте его удобно вынести в отдельную DI-квалификацию для тестирования.
